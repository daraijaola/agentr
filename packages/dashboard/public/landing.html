Isolation & security

Every user, fully isolated.

One platform deployment serves unlimited users. Each account gets its own wallet, workspace, and agent. Nothing crosses tenant boundaries.

Private workspace
Path-isolated sandbox
Every user's files live under /sessions/{tenantId}/ — path traversal is blocked at the API level. Your agent cannot access anyone else's data.
TON wallet
Auto-generated on signup
Each user gets a unique, isolated TON wallet the moment they sign up. Funds and transactions are fully contained within your tenant.
Process isolation
PM2 namespaced processes
Deployed bots and apps run under per-tenant PM2 namespaces. Processes from one user cannot interfere with or access another user's deployments.
Identity
Immutable core files
Your agent's SOUL, IDENTITY, and STRATEGY files cannot be modified by the agent at runtime. Your agent's values are set by you, not rewritten by it.
